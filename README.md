# Archpack

Archpack is a small directory-pack tool.

The MVP core is:

```text
pack directory → file tree → explicit repair
```

A pack is a directory that contains a `tree/` directory. Archpack copies files from `tree/` into an output directory and can explicitly repair missing or overwritten output files from the same pack.

Archpack does not continuously enforce a structure and does not monitor projects in the background.

## Demonstration

Step-by-step demo pack and scripts:

```text
demo/
```

One command from the repository root:

```powershell
.\demo\run.ps1
```

```bash
bash demo/run.sh
```

See `demo/README.md` for the Taskflow sample app, per-directory `AGENTS.md`, and what each step deploys into `demo/workspace/`.

## Quick start

Install for local development:

```bash
python -m pip install -e ".[dev]"
```

Generate files:

```bash
archpack unpack examples/minimal-pack --out tmp/out
```

Add only missing files without stopping on existing files:

```bash
archpack unpack examples/minimal-pack --out tmp/out --skip-existing
```

Repair missing files:

```bash
archpack repair examples/minimal-pack --out tmp/out
```

Overwrite existing files only when explicitly requested:

```bash
archpack repair examples/minimal-pack --out tmp/out --overwrite
```

Generate scoped AGENTS.md files with the reviewed agents plugin:

```bash
archpack agents-generate examples/agents-pack --out tmp/agents-out
```

Run tests:

```bash
pytest
```

## Pack layout

```text
examples/minimal-pack/
└─ tree/
   └─ README.md
```

Only files under `tree/` are generated by core commands.

## Agents plugin

The first reviewed plugin is the AGENTS.md generator.

AGENTS.md generation is managed from one source file:

```text
agents.toml
```

Generated `AGENTS.md` files are derived outputs. Edit `agents.toml`, then regenerate.

It reads:

```text
examples/agents-pack/agents.toml
```

and writes scoped `AGENTS.md` files such as:

```text
AGENTS.md
src/AGENTS.md
src/services/AGENTS.md
```

Scoped means each generated `AGENTS.md` contains only the local rules for that directory. Non-root generated files include a short navigation hint to read the parent `AGENTS.md` first.

Each local rule unit is limited to 30 rules.

The plugin is explicit. It does not run during `unpack` or `repair`.

See:

```text
docs/plugins/agents.md
```

## Current core boundary

Core:

- read a pack directory
- read files under `tree/`
- generate directories
- generate files
- write file contents
- skip existing files when requested
- explicitly repair generated files from the pack when requested

Core does not:

- continuously enforce the structure
- monitor the project in the background
- auto-repair without an explicit user action
- generate `AGENTS.md` as special behavior
- run plugins implicitly
